System Architecture

INPUT LAYER

  • Student client input (draft / publish)
  • Teacher dashboard actions
  • Session metadata (group, topic, role)

STATE & LOGIC LAYER

  • Front-end session state (draft, sent, acknowledged)
  • Prompt-response handler for AI scaffolding
  • Nudge logic (silence threshold / stuck detection)

OUTPUT LAYER

  • Public round-table thread updates
  • Participation analytics aggregation
  • Teacher notifications and equity report

Data Handling Evidence

Each student interaction updates local state and is mirrored to session logs used by analytics cards (talk-time ratio, silent-window, contribution count).

Implementation Scope

Alpha uses front-end mock data plus deterministic rules to simulate AI nudges while preserving the same state transitions planned for production APIs.

V1.0 Low-Fi Insights

Initial testing showed that students felt 'over-monitored' by the AI. We pivoted to a 'Pull' model where the AI only intervenes when requested or during prolonged silence.

V2.0 High-Fi Refinement

Added the 'Sanctuary Mode'—a private drafting space that uses Arthurian metaphors to make the technology feel more like a supportive companion than a rigid tool.
